## 🖥 Code convention
 
- **naming rule**

```
1. 변수, 함수: 카멜케이스
    - var 금지
2. 상수: 대문자 SNAKE_CASE
3. 폴더명 : 모두 소문자
4. 파일이름 : 카멜케이스
```

- **git commit message rule** 
```
- master : 제품으로 출시될 수 있는 브랜치
- dev : 다음 출시 버전을 개발하는 브랜치
- feature : 기능을 개발하는 브랜치
- release : 이번 출시 버전을 준비하는 브랜치
- hotfix : 출시 버전에서 발생한 버그를 수정 하는 브랜치
```
